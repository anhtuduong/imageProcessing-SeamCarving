W03H03 - Seam Carving

Happy and Snappy recently went on vacation. Snappy took some pictures of Happy. But the individual elements of the pictures are too far apart for Happy. You should now write a small image editing program that corrects this.

Luckily, after some searching, you found an algorithm that allows you to edit the images as you wish: Seam Carving.


(Here one of the pictures of Happy)

General information

  • There are some useful methods for working with arrays in java.lang.System and java.util.Arrays .
  • The arrays passed are always at least as long as specified by the other parameters. For example, if a length and width are specified, the associated array has at least that lengthLawithBroad\text{Length}\cdot\text{Breite}LawithBroad. But it can also be larger.
  • Images are presented as a one-dimensional array. The image is saved line by line. In practice, this looks like this:

    The origin is at the top left, the X axis increases to the right and the y axis decreases.
  • All images always have a size and length0\geq00
  • If earlier methods are to be reused for the implementation, these are tested a second time if incorrect results are returned, with the methods used being replaced with correct versions. As a result, correctly implemented methods that use faulty required methods are recognized as correct.

The Algorithm

The rough functionality is now explained with the help of this picture

To process your images with Seam Carving you need to implement 3 essential steps:

  1. Find more important and less important pixels. You can use different algorithms for this, we will use the gradient magnitude for this weighting. More important pixels are assigned a larger value.

  2. Generate the eponymous seams from these weights. These are basically lines running through the image that are attracted to regions pre-determined to be unimportant, moving laterally by a maximum of 45. The weight of the individual pixels through which the line goes is added up.

  3. Keep removing the least important seams until the desired width is achieved.

(Source of images: Wikipedia / Newton2 )

picture format

The images are stored as a one-dimensional array of ints. Each int represents a pixel in rgb format. A black pixel looks like this:
0xaa000000agbr\begin{array}{|c|c|c|c|c|}\hline0x& \alpha & g & b & r\\\hline\end{array}0x _a aa00g00b00r
The one witha\alphaaThe area described contains the transparency and is ignored by us here.

Gradient Magnitude

Gradient Magnitude measures the change in the local color gradient in the x or y direction and is therefore larger in more complex image areas. It is defined as follows:
abcdefghi\begin{array}{|c|c|c|}\hline a & b & c \\ \hline d&e&f \\\hline g&h&i\\\hline\end{array}adgbehcfi
At e, for one color band (the r,b and g channels), is the magnitude(bh)2+(df)2(b-h)^2+(d-f)^2(bh)2+(df)2, where we add the magnitudes of the 3 bands.

However, values relevant for the calculation are missing at the edges of the image. We set all pixels for which this formula does not provide a value to the maximum value of an ints. An example of such a point would be the following:
x\begin{array}{|c|c|}\hline &amp; \\ \hline x&amp; \\\hline &amp;\\\hline\end{array}x
The point x borders one of the edges of the image, which prevents us from calculating the weight of the point. Therefore we set x to the maximum value of , as described int.

methods

The calculation is divided into 2 steps.

  1. computeGradientMagnitude This method gets 2 pixels as an integer parameter and calculates the gradient magnitude between them using the well-known formula. The entire gradient magnitude over 4 is not calculated, but only for 2 opposite pixels. So you always have to add 2 calls to this method to calculate the full magnitude.
  2. togradientMagnitude This method calculates the gradient magnitude as described and writes the results into the 2nd passed array ( gradientMagitude).

Manual fixes

Sometimes it happens that you want to keep certain parts of the image, which, however, have little relevance for the weighting algorithm used. It therefore makes sense to be able to overwrite this manually.

This is implemented by giving a mask in addition to the image to be processed. In it, each pixel that is to be kept is marked with black (all 3 channels at 0).
The weight of these pixels should Integer.MAX_VALUEget the maximum value. This is implemented in combineMagnitudeWithMask.

One can assume that both arrays are large enough.

Seams

The seams are lines through the image that contain exactly one pixel from each line and run through the image with a maximum inclination of 45.

We build up the seams iteratively row by row. A seam begins at each point of the first image line. We determine the further course of the seam as follows:

  • From the 3 pixels below the previous pixel, the one with the lowest weight is chosen.

    If the left and right pixel have the same weight, the left one is chosen.

    If the middle one is one of the minimum weight pixels, it is always chosen.

This is done with each seam in each row, adding up the weights and storing x positions of the points. The weights are stored in an longarray and the points in a two-dimensional intarray. The outer array is indexed with the x position of the start pixel of the seam and the inner one with the y coordinate.

All the required pixels may not be present at the edges of the image. Of course, the seam cannot go beyond the edge. Accordingly, there are fewer options left for the next pixel.

This is implemented in buildSeams.

Remove Seams

Now the last step is missing: removing a seam from the image. As parameters you get an array with all x coordinates of the seam, an image array and the current height and width. Your task is to shift the pixels in this array so that a 1 pixel narrower image is left with the seam pixels removed. The result should be in the image array passed as a parameter. The size of the array cannot and does not have to be adjusted. This is implemented in removeSeam.

Alltogether

It is now time to combine the previous functionality. The number of seams to remove is given by the difference of widthand newWidth. The seam that has the minimum weight is always removed. If several have the same weight, the first (smallest index) is chosen. For each seam to be removed, the gradient magnitude must be recalculated, combined with the mask, and the seams recalculated. The result should be returned at the end with an adjusted length.
